## 1. Introduction / Overview

사용자가 제공한 시나리오(후츠릿 영상)와 동일하게 **"인간은 승인만 하고, AI 팀이 24시간 연구→작성→배포 준비"** 를 수행하는 자율 AI 마케팅/콘텐츠 팀을 구축한다.

이 프로젝트는 다음을 동시에 진행한다:
- Hermes Agent (또는 Langflow + 로컬/클라우드 모델) 기반의 **지속 실행 AI 에이전트 팀** 구축 (VPS 또는 서버에서 항상 동작)
- 이 AI 팀을 활용하거나, 이 시스템 자체를 홍보하기 위한 **마케팅 웹사이트 + 리드 생성 풀 퍼널** 구축
- 실제 콘텐츠 에셋 대량 생산 + 광고/퍼포먼스 세팅 + 분석 체계 완비

**결과물**: 1~3개월 내에 "승인만 하면 콘텐츠가 자동으로 올라가는" 운영 가능한 AI 마케팅 시스템 + 이를 런칭하는 공개 마케팅 자산 일체.

---

## 2. Goals (구체적 목표)

- 24시간 동작하는 AI 콘텐츠/마케팅 팀 (research → draft → human approval gate → publish) 완성
- Supabase 중심의 리드 수집·관리·CRM·콘텐츠 캘린더·승인 큐·성과 추적 시스템 구축
- 고품질 마케팅 랜딩 페이지 / 사이트 제작 (전환 최적화, SEO, A/B 테스트 가능)
- 초기 콘텐츠 자산 대량 생산 (블로그, 소셜, 이메일 시퀀스, 스크립트 등)
- Meta/Google 등 퍼포먼스 광고 기본 세팅 + 추적 코드 + 대시보드
- 전체 과정을 문서화하여 "이 시스템으로 실제 런칭한 사례"로 활용 가능하게 만들기
- 중기(1~3개월) 안에 실제 운영/측정 가능한 상태 도달

## 3. Target Audience (타겟)

**1차 타겟 (시스템 구매자/사용자)**
- 1인 창업자, 소규모 팀, 콘텐츠 마케터
- AI 자동화에 관심 있는 개발자/기획자 (Hermes, Langflow, n8n, Make 등 사용자)
- 기존 제품(AI 사주, 꿈해몽, 스토리캐스트 등)을 효율적으로 마케팅하고 싶은 사람

**2차 타겟 (이 시스템으로 마케팅할 최종 고객)**
- 한국의 AI 도구, 생산성, 점술/운세, 교육, 콘텐츠 크리에이터 관련 관심층
- 25~45세, 온라인으로 정보 탐색하고 구매하는 층

**채널**: YouTube, 블로그, X(트위터), Instagram/Threads, 이메일, SEO

## 4. User Stories

### US-001: AI 콘텐츠 팀 기본 아키텍처 구축 (Hermes/Langflow)
**Description:** 개발자/기획자로서, VPS 또는 로컬 서버에서 항상 실행되는 Hermes 스타일 AI 에이전트 팀을 세팅하고 싶다. 연구 → 초안 → 승인 대기 → 배포 준비까지 자동으로 흐르길 원한다.

**Acceptance Criteria:**
- [ ] Hermes Agent (또는 동등한 Langflow 워크플로우) 설치 및 VPS 영구 실행 환경 구성 가이드 + 실제 세팅
- [ ] 콘텐츠 생성을 위한 전문 에이전트 프로필 정의 (Researcher, Writer, Editor, Publisher, SEO Analyst 등)
- [ ] 승인 게이트 (Discord, 이메일, 또는 웹 UI) 구현 — 인간이 "승인"만 하면 다음 단계 진행
- [ ] 생성된 콘텐츠와 메타데이터를 Supabase에 저장 (캘린더, 상태, 버전 관리)
- [ ] 기본 모니터링 (로그, 실행 기록) 제공

### US-002: 마케팅 랜딩 페이지 / 웹사이트 제작
**Description:** 잠재 고객이 "이 AI 마케팅 팀이 실제로 뭐하는지"를 30초 안에 이해하고, 리드를 남길 수 있는 고전환 랜딩/사이트가 필요하다.

**Acceptance Criteria:**
- [ ] 현대적이고 설득력 있는 디자인 (프론트엔드 스킬 또는 직접)
- [ ] 주요 기능 데모/스크린샷/비디오 섹션
- [ ] 리드 캡처 폼 (이름, 이메일, 관심사) → Supabase 저장 + 즉시 감사 이메일/자동화
- [ ] SEO 최적화 + 메타 태그 + OG 이미지
- [ ] A/B 테스트 가능한 구조 (적어도 1~2개 변형 준비)
- [ ] 모바일 완벽 대응
- [ ] "지금 시작하기" 또는 "데모 요청" CTA 강력 배치

### US-003: Supabase 기반 리드 + 콘텐츠 운영 시스템
**Description:** 모든 리드, 콘텐츠 일정, 승인 상태, 발행 기록, 광고 성과를 한 곳에서 관리하고 싶다.

**Acceptance Criteria:**
- [ ] 테이블 설계: leads, content_calendar, approvals, published_posts, campaigns, analytics_events 등
- [ ] RLS 정책 적절 설정 (인증 사용자만 접근)
- [ ] 간단한 관리 대시보드 (또는 관리용 페이지) — 리드 목록, 콘텐츠 상태 보기, 승인 액션
- [ ] Edge Functions 또는 트리거로 승인 시 자동 publish 액션 트리거 가능
- [ ] 이메일 자동화 연동 포인트 (Resend, SendGrid, 또는 Supabase + 기타)

### US-004: 콘텐츠 에셋 대량 생산 파이프라인
**Description:** 실제로 사용할 수 있는 초기 콘텐츠를 AI 팀이 생성하고, 인간 승인 후 배포할 수 있어야 한다.

**Acceptance Criteria:**
- [ ] 주제 선정 (타겟 제품/브랜드에 맞춘 10개 이상 주제 풀)
- [ ] 블로그 포스트 초안 5~8개 생성 + 승인 플로우 통과
- [ ] 소셜 포스트 (X, Threads, LinkedIn 등) 20개 이상 배치
- [ ] 이메일 nurture 시퀀스 5~7통
- [ ] 모든 콘텐츠가 Supabase content_calendar에 등록 + 상태 관리
- [ ] 실제 1개 이상 채널에 수동 또는 자동 배포 성공 사례 기록

### US-005: 광고/퍼포먼스 세팅 + 추적 인프라
**Description:** Meta, Google Ads 등에서 광고를 집행할 때 성과를 정확히 측정하고 대시보드에서 볼 수 있어야 한다.

**Acceptance Criteria:**
- [ ] 픽셀/전환 API 기본 설치 가이드 (랜딩 페이지에)
- [ ] UTM 템플릿 및 캠페인 구조 문서화
- [ ] Supabase에 광고 이벤트/비용/전환 데이터 수집 포인트
- [ ] 간단 성과 대시보드 (노출, 클릭, 비용, 리드, CPA 등)
- [ ] 초기 테스트 광고 1~2개 세팅 (또는 완전 가이드)

### US-006: 분석 + A/B 테스트 + 개선 루프
**Description:** 어떤 콘텐츠/랜딩이 효과적인지 데이터로 보고, 빠르게 개선할 수 있는 체계를 갖추고 싶다.

**Acceptance Criteria:**
- [ ] Supabase + 간단한 시각화 (또는 Metabase/자체 페이지)로 주요 KPI 대시보드
- [ ] 최소 1개 이상 A/B 테스트 실행 (헤드라인, CTA, 또는 콘텐츠 형식)
- [ ] 주간/월간 리뷰를 위한 리포트 자동 생성 또는 수동 템플릿
- [ ] AI 팀이 성과 데이터를 읽고 다음 콘텐츠 아이디어를 제안하는 기능 (옵션)

## 5. Functional Requirements (FR)

**FR-1.** Hermes (또는 Langflow 기반) 에이전트가 서버에서 지속 실행되어야 한다 (노트북 꺼도 동작).
**FR-2.** 인간 승인 게이트가 명확히 존재해야 하며, 승인 전에는 외부 배포가 일어나지 않아야 한다.
**FR-3.** 모든 핵심 데이터(리드, 콘텐츠, 승인, 발행, 캠페인)는 Supabase에 저장된다.
**FR-4.** 랜딩 페이지는 최소 60% 이상 전환을 목표로 설계 (벤치마크는 실제 측정 후 개선).
**FR-5.** 콘텐츠 생성 → 승인 → 배포까지의 전체 흐름이 최소 1회 이상 실제 동작해야 한다.
**FR-6.** 광고 추적 및 리드 소스 attribution이 가능해야 한다.
**FR-7.** 시스템은 확장 가능하게 모듈화 (추후 다른 제품에도 적용 가능).
**FR-8.** 전체 구축 과정과 운영 매뉴얼이 문서화되어 있어야 한다.

## 6. Non-Goals (Out of Scope, 이번 단계에서는 하지 않을 것)

- 완전한 무인 (zero human touch) 배포 (승인 게이트는 반드시 유지)
- 대규모 광고 집행 및 예산 소진 (초기 테스트 수준)
- 복잡한 CRM 영업 파이프라인 (기본 리드 + nurture까지만)
- 다국어 풀 지원 (한국어 우선, 영어는 추후)
- 기존 모든 유저 프로젝트를 동시에 마케팅
- 고도화된 추천/개인화 엔진 (1차는 규칙 + 간단 프롬프트 기반)

## 7. Design & UX Considerations

- 전문적이면서도 "AI가 진짜 일하는 느낌"을 주는 모던 디자인
- Hermes / Langflow 친화적인 기술적 투명성 강조 (아키텍처 다이어그램, 에이전트 역할 설명)
- 신뢰 형성: 실제 동작 스크린샷, 로그 예시, "승인 전/후" 비교
- 모바일 최우선 (많은 타겟이 모바일)

---

## 8. Technical Considerations

- **Backend / Data**: Supabase (PostgreSQL + Auth + Storage + Edge Functions + Realtime)
- **AI Agent Layer**: Hermes Agent (VPS 배포) 또는 Langflow Desktop + 로컬 모델 (gemma3:4b 등) + 클라우드 모델 혼합
- **Frontend**: Next.js (App Router) 또는 Vite + React (성능/배포 편의 고려). Vercel 또는 Supabase Hosting
- **Content Pipeline**: Markdown 중심 + 구조화된 JSON 메타데이터
- **승인 인터페이스**: Discord 봇 / 간단 웹 대시보드 / 이메일 (최소 1개)
- **추적**: Meta Pixel, Google Tag Manager, UTM, Supabase 이벤트 수집
- **배포**: Git + CI (GitHub Actions 추천) 또는 수동

**중요 제약**:
- 노트북을 항상 켜두지 않을 수 있음 → VPS 또는 항상 켜진 서버 필요
- 비용 효율적 (월 1~3만 원대 유지 가능 목표, 영상 참고)

## 9. Success Metrics (KPI)

**시스템 구축 완료 기준 (MVP ~ 중기)**
- AI 팀이 최소 1개 이상 콘텐츠를 "연구 → 초안 → 인간 승인 → 저장/배포 준비"까지 완료
- 랜딩 페이지 실제 배포 + 최소 50명 이상 방문 + 5명 이상 리드 수집 (테스트 기간)
- Supabase에 leads 10+ 건, content_calendar 15+ 건, approvals 기록 존재
- 주요 KPI 대시보드에서 데이터 확인 가능
- 문서화된 운영 가이드 (README + PRD + 매뉴얼) 완성

**캠페인 성과 (1~3개월 내)**
- 리드 획득 비용 (CPL) 목표 설정 후 실제 측정
- 콘텐츠 발행 주기: 주 3~5회 이상 안정화
- 이메일 오픈율 / 클릭율 측정 및 개선
- 전체 퍼널 전환율 가시화

## 10. Open Questions / Assumptions

- **제품/브랜드 명확화 필요**: 이 AI 마케팅 시스템 자체를 판매할 것인가, 아니면 특정 기존 프로젝트(예: 사주/스토리/AI 툴)를 이 시스템으로 마케팅할 것인가?
- Hermes Agent vs Langflow + 커스텀 에이전트: 어떤 스택을 주력으로 할지 최종 결정 필요 (비용/안정성)
- 승인 채널: Discord가 편한가, 아니면 전용 웹 대시보드가 더 좋은가?
- 첫 타겟 제품: 영상 시나리오를 따라하면서 구체 제품 하나를 정해서 파일럿 런칭할지
- 예산: VPS 비용, 광고 테스트 비용, 도메인/이메일 서비스 비용
- 팀: 1인 (사용자 + AI)으로 진행하는가, 추가 도움 받을 부분은?
